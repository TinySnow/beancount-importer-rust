# 开发与调试手册

## 1. 本地开发流程

推荐流程：

1. 修改配置或代码
2. `cargo fmt`
3. `cargo test --quiet`
4. 用真实/脱敏样本执行一次 `cargo run`
5. 用 Beancount 工具链做语法与平账检查

## 2. 日志与诊断

支持参数：

- `--log-level trace|debug|info|warn|error`
- `-v` 等价 debug
- `-q` 等价 error

常见诊断点：

- 读取文件编码是否正确
- 表头是否与 mapping 对齐
- 规则条件是否命中
- 账户覆盖优先级是否符合预期

## 3. 常见问题排查

### 3.1 日期被引号包裹

检查 `output.date_format` 是否写成带引号的字符串；项目内部已做去引号兼容。

### 3.2 metadata key 非法

检查是否存在中文/空格 key；建议通过 mapping 输出标准英文 key。

### 3.3 证券卖出不平

确认是否存在：

- 卖出持仓行（含 `{}` 推断成本）
- 现金行
- 手续费行（如存在）
- PnL 占位行

### 3.4 回购交易金额异常

确认回购是否按固定面值建模，且开仓/到期逻辑成对出现。

## 4. 测试策略建议

### 4.1 单元测试

覆盖以下函数：

- metadata 归一化
- 规则匹配操作符
- 证券符号/币种规范化
- writer 输出关键语法

### 4.2 集成测试

使用脱敏样本文件验证：

- 不同 provider
- 大额数据量
- 非 UTF-8 编码输入

## 5. 代码风格建议

- 核心逻辑禁止 `unwrap/expect`
- 新增 provider 不要把业务分支塞入公共模块
- 复杂规则优先放配置，不要硬编码
- 账户命名保持层级稳定，避免频繁重命名
