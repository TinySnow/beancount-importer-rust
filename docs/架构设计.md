# 架构设计

## 1. 总体流程

项目采用“读取 -> 归一化 -> 规则匹配 -> 分录生成 -> Beancount 输出”的流水线：

```text
账单文件(CSV/XLSX)
  -> CsvRecordReader
  -> RawRecord
  -> RuleEngine(全局规则 + provider 规则)
  -> Provider::transform
  -> Transaction
  -> BeancountWriter
  -> .beancount
```

## 2. 模块分层

### 2.1 `runtime`

- `src/runtime/config_loader.rs`
  - 负责加载并合并 `global/provider/mapping` 配置。
  - 处理 `mapping_file` 相对路径与 fallback 路径。
- `src/runtime/pipeline.rs`
  - 负责转换主流水线：`Provider::transform`、strict 行为、统计日志。
- `src/runtime/sorting.rs`
  - 输出前排序：先成交日期，再委托日期（同日升序）。
- `src/runtime/inventory/*`
  - lot 补全后处理：seed 预加载、卖出 lot 匹配、分录拆分回写。
- `src/runtime/pnl.rs`
  - 逐笔写入 `grossPnl/feeTotal/netPnl` 元数据。
- `src/runtime/mod.rs`
  - 负责入口编排与 I/O（读源文件、写目标文件）。

### 2.2 `interface`

- `src/interface/provider.rs`
  - 定义 `Provider` trait：`name/description/parse/transform`。
  - 默认 `parse` 复用 `CsvRecordReader`，支持 strict_mode 透传。

### 2.3 `model`

- `src/model/data/raw_record.rs`：统一中间记录模型。
- `src/model/rule/*`：规则定义、条件匹配、命中结果合并。
- `src/model/transaction/*`：Beancount 交易与过账模型。
- `src/model/reader/csv_reader/*`：CSV/XLSX 读取、表头与字段映射。
- `src/model/writer/beancount_writer/*`
  - `beancount_writer.rs`：写出入口。
  - `directives.rs`：`open/commodity` 指令写出。
  - `render.rs`：交易正文与 metadata 渲染。

### 2.4 `providers`

- `src/providers/third_party/*`：支付宝/微信/京东/美团。
- `src/providers/banks/*`：工商银行/建设银行。
- `src/providers/securities/*`：富途/银河证券。
- `src/providers/shared/cashflow/*`
  - 通用收支模型：支出/收入双分录生成。
- `src/providers/shared/securities/*`
  - 证券模型：普通买卖、逆回购、银证转账。
  - 支持 `securities_accounts` 与兼容 `default_*` 的分层回退。

### 2.5 `utils`

- 编码识别与文本解码。
- 金额解析与 `transform`（如 `abs/negate`）。
- metadata key 归一化与合法化。

## 3. 规则引擎设计

### 3.1 执行顺序

每个规则集合内部排序规则为：
1. `priority` 升序（小值先执行）。
2. `specificity` 升序（条件更少的先执行）。
3. 文件书写顺序。

整体顺序为：`global_rules` 先执行，provider `rules` 后执行。

### 3.2 结果合并

`MatchResult` 合并策略：
- 单值字段（账户、payee、narration、flag）：后命中覆盖先命中。
- `metadata`：按 key 合并；同 key 后命中覆盖先命中。
- `tags/links`：追加。
- `ignore`：一旦为 `true` 持续为 `true`。

### 3.3 行为控制

- `terminal: true`：命中后立即停止后续规则。
- `ignore: true`：该记录最终不输出分录。

## 4. 现金流与证券两条转换链

### 4.1 现金流链（bank / third-party）

- 判定收支方向：优先看 `transaction_type` 关键词，回退到金额符号。
- 支出：借费用、贷资产。
- 收入：借资产、贷收入。

### 4.2 证券链（securities）

- 普通买入：持仓增加、券商现金减少，差额入 `fee` 或 `rounding`。
- 普通卖出：持仓减少、券商现金增加，补 `fee` 与 `pnl` 承接行。
- 逆回购：固定面值建模（每份 100 CNY），到期差额计利息或费用。
- 银证转账：仅资产账户间划转，不走 `pnl/rounding`。

## 5. 输出层设计

`BeancountWriter` 关键行为：
- 交易按日期排序输出，结果稳定。
- 自动写出 `commodity`（非法币且涉及持仓/价格信息时）。
- 可选写出 `open`（`emit_open_directives=true`）。
- metadata 按 key 排序，输出可比较。

## 6. 扩展点

新增 provider 时通常只需：
1. 实现 `Provider` trait。
2. 在 `src/providers/mod.rs` 注册。
3. 提供 `config/<provider>.yml` 与 `mapping/<provider>.yml`。

具体步骤见：[供应商扩展指南](供应商扩展指南.md)。
