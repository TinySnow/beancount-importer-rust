# 配置详解

本文档给出本项目全部核心配置项及推荐写法。

## 1. provider 配置结构

典型文件：`src/config/<provider>.yml`

```yaml
name: wechat
mapping_file: src/mapping/wechat.yml

default_asset_account: Assets:Digital:WeChat:Balance
default_expense_account: Expenses:Others
default_income_account: Income:Others
default_currency: CNY

default_fee_account: Expenses:Investing:Fees
default_pnl_account: Income:Investing:Capital-Gains
default_rounding_account: Expenses:Investing:Rounding

csv_options:
  delimiter: ','
  quote: '"'
  flexible: true
  encoding: auto

skip_header_lines: 0
has_csv_header: true

output:
  date_format: "%Y-%m-%d"
  decimal_places: 2
  emit_open_directives: false

rules: []
```

## 2. csv_options

- `delimiter`：分隔符
- `quote`：引号字符
- `flexible`：是否允许列数不齐
- `encoding`：`auto/utf-8/gbk/...`
- `comment`：注释行前缀（可选）

## 3. output

- `date_format`：日期格式
- `decimal_places`：金额小数位
- `account_prefix`：账户统一前缀（可选）
- `emit_open_directives`：是否自动输出 `open`
- `open_date`：指定 `open` 日期（可选）

## 4. rule 定义

```yaml
- name: 餐饮支出
  conditions:
    - field: type
      equals: 支出
    - field: peer
      regex: 小吃|餐饮|火锅
  action:
    debit_account: Expenses:Food
    credit_account: Assets:WeChat:Balance
    metadata:
      scene: daily
  priority: 10
  terminal: false
```

### 4.1 可用操作符

- `equals`
- `contains`
- `regex`
- `starts_with`
- `ends_with`
- `greater_than`
- `less_than`
- `between`
- `in`
- `not_empty`
- `is_empty`

### 4.2 匹配模式

- `match_mode: and`（默认）
- `match_mode: or`

### 4.3 action 字段

- `debit_account`
- `credit_account`
- `narration`
- `payee`
- `ignore`
- `metadata`（键值对）
- `tags`
- `links`
- `fee_account`
- `pnl_account`
- `rounding_account`

## 5. mapping 配置

典型文件：`src/mapping/<provider>.yml`

```yaml
date: "交易时间"
payee: "交易对方"
narration: "商品"
transaction_type: "收/支"
amount:
  column: "金额(元)"
  transform: abs

extra_fields:
  peer: "交易对方"
  peerAccount: "对方账号"
  orderId: "交易单号"
  txType: "交易类型"
  method: "支付方式"
```

### 5.1 简写语法

若字段仅需 `column`，可简写为：

```yaml
payee: "交易对方"
```

等价于：

```yaml
payee:
  column: "交易对方"
```

### 5.2 详细语法

当需要转换时使用对象语法：

```yaml
amount:
  column: "金额"
  transform: abs
```

## 6. metadata key 对照

推荐直接在 mapping 里使用最终输出 key：

- `peer`
- `peerAccount`
- `source`
- `txType` / `category`
- `type`
- `payTime`
- `method`
- `orderId`
- `merchantId`

## 7. 账户覆盖优先级

以手续费账户为例：

1. 规则级 `action.fee_account`
2. provider 级 `default_fee_account`
3. provider 级 `default_expense_account`
4. 内置默认 `Expenses:Investing:Fees`

PnL 与 rounding 同理。
