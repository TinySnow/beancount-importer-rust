# 供应商扩展指南

本指南描述如何从零新增一个供应商。

## 1. 新增代码文件

建议放置位置：

- 银行：`src/providers/banks/<name>.rs`
- 第三方：`src/providers/third_party/<name>.rs`
- 证券：`src/providers/securities/<name>.rs`

## 2. 实现 Provider trait

```rust
pub struct XxxProvider;

impl Provider for XxxProvider {
    fn name(&self) -> &'static str { "xxx" }

    fn description(&self) -> &'static str { "Xxx statement importer" }

    fn transform(
        &self,
        record: RawRecord,
        rule_engine: &RuleEngine,
        config: &ProviderConfig,
    ) -> ImporterResult<Option<Transaction>> {
        transform_cashflow_record("xxx", "Assets:Xxx", record, rule_engine, config)
    }
}
```

说明：

- 通用收支类建议直接复用 `transform_cashflow_record`
- 证券类建议参考 `futu.rs` 单独实现

## 3. 注册 provider

在 `src/providers/mod.rs` 中注册：

```rust
registry.register(Arc::new(third_party::xxx::XxxProvider));
```

并在对应 `mod.rs` 暴露模块。

## 4. 提供配置文件

- `src/config/<name>.yml`
- `src/mapping/<name>.yml`

原则：

- 字段映射不硬编码在 Rust 代码中
- 规则尽量表达业务语义
- 账户命名保持统一层级

## 5. 测试建议

至少覆盖：

- 基础导入是否成功
- 关键规则是否命中
- metadata key 是否符合预期
- 证券交易是否平账
- 输出 Beancount 可解析

## 6. 迁移兼容建议

如从其它项目迁移：

1. 先保持老规则逻辑不变
2. 优先改 mapping key 到最终 metadata key
3. 再逐步精简规则与账户层级

## 7. 提交前检查

```bash
cargo fmt
cargo test --quiet
```

必要时再跑一次真实样本导入手工检查输出。
